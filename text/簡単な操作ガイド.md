1.ログインの流れ

```
./ircserv 4444 pass (ポート番号は 4242 でもいいですし、パスは3文字以上の重複と4文字以上です。)
```

上記のような構成で立ち上げたとします。

2.irssi でログインする。

irssi を立ち上げると
以下の流れでログインできます。

```
/connect localhost 4444
/quote pass pass
```

コマンドは大文字でも小文字でも構いません。
ipv6 のときは、<localhost>を<::1>にしてください

irssi は、ユーザーとニックネームを勝手に取得していまうので、ニックネームが重複するとアンダーバーをつけて来ます。
事前にニックネームを指定しないとうまく挙動しないので。
立ち上げのときに

```
irssi -n name -c localhost -p 4444 -w "pass"
```

とします。そうするとニックネームの指定ができて、一発でログインもできます。

3.nc でのログイン

irssi は、勝手にニックネームとユーザーネームを補ってくれますが、nc を使う場合は、そうは行きません。

```
nc localhost 4444
PASS pass
NICK name
USER tanaka 0 * :tanaka
```

でログインをしてください。USER の第 2、3 引数は、チェックしてないので、なんでもいいです。

4.ログイン後
ログインをすると、画面にウェルカムメッセージが出るはずです。

5.チャンネルにログインする。
irssi で、コマンドを打つ際は、/で始めます。

```
/join #test
```

nc では

```
join #test
```

これで join コマンドとチャンネル名が指定できたはずです。

このあと自動的にチャンネル画面に遷移します。

6.チャンネルメッセージを打つ。
irssi で、画面が自動的に遷移した場合は、コマンドライン上でコメントを打つと自動的に privmsg として処理されます。

```
こんにちわ
```

と打つだけで、画面に反映されるのを確認できるはずです。

一方、nc では、アンノウンコマンド認定されるので、コマンドを補いましょう

```
PRIVMSG #gen :こんにちわ
```

のように、コマンド、対象、トレーリングのように入力します。
メッセージの前に、:を補う必要があるのが注意点です。

7.クライアント同士で会話する。
複数のユーザーでチャンネルにログインします。
しかし、個別に会話したい場合は、このように会話ができます。

```
/msg 相手のニックネーム名 メッセージ

//具体的にすると

/msg otsuki チンリロしようぜ！
```

のようにメッセージを打ちましょう。
（nc では、PRIVMSG 　 nick :msg 　のようにする必要があります。irssi と nc の操作の違いはこの時点である程度把握されたはずです）

8.画面遷移
irssi 上では、画面の切り替えは、キーボードのコントロール N で行います。（P で逆方向に戻ります）

さきほどの画面で、画面を切り替えると、個人間メッセージの画面に移動できるはずです。
このように、チャンネル以外で個別ユーザーと通話をすることができます。

9.オペレーター権限

チャンネルを作成したユーザーが、オペレーター（管理者）になります。
オペレーターはユーザーの追放や、モードの設定ができます。

```
/mode +i
```

とすると、その操作しているチャンネルをインバイトモード（招待制）にすることができます。
たとえば、他のユーザーでチャンネルに/join #test のようにログインしようとしてもできないはずです。
これに対応するのが invite コマンドです。-k でジョインにパスワードが必要にすることもできます。

10.bot
現段階で、bot が 5 つあります。

irssi で、

```
!h
!o
!c
!b
!s
!r
```

と入力すると、ボットが対応してくれます。
!h は、挨拶、!o はおみくじ、!c は、チンリロ、!b と!s は、簡易的なブラックジャックです。
!r は、その中にいる人をランダムで選出します。
