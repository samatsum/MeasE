・IRC サーバー
全体のリソースを管理する。（バッファーの取得処理と、チャンネルとクライアントの所有）
メンバ：サーバーデータ、ソケット IO と、チャンネルと、クライアントを”所有”する。
API ：クライアントの登録削除　チャンネル生成、探索、削除、チャンネル横断機能（QUIT などの全体ブロードキャストに使う）

・コマンドハンドラー
コマンド解析と、解析に基づいたルーティングを提供する。
メンバ：IRC サーバーの参照（必要なメソッドを呼び出すため）
API 　：パースコマンド（サーバーで発生したイベントの、解析へのエントリーのみ）

・クライアント
ここの接続、登録状態、参加チャンネルなど、クライアントに関する情報を保有し操作に必要な API をコマンドハンドラ提供する。
メンバ：個別の必要情報
API：ゲッターセッターがメイン。IRC サーバーが、もってるバッファーの終端確認するためのコマンド抽出も。

・チャンネル
